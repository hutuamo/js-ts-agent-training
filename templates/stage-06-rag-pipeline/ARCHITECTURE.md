# 架构说明 - Stage 06 RAG Pipeline

## 核心理念

把检索当作数据流水线加有据可查的答案步骤来对待。

## 核心组件

1. 语料库定义
2. 摄入和归一化
3. 分块和元数据
4. 索引和检索
5. 有据可查的答案生成
6. 引用或 provenance 处理
7. 检索评估集

## 边界规则

- 语料库质量在检索调优前应该可检查
- 检索应该能独立于生成来测试
- 最终答案应识别它们的证据
- 不支持的声明应被拒绝或清晰标记
- 记忆和检索应保持分离概念

## 推荐流程

1. 摄入源材料
2. 归一化并分块
3. 附加元数据
4. 索引并检索
5. 检查检索到的证据
6. 生成有据可查的响应
7. 针对带标签查询记录成功或失败

## 需要计划的失败类别

- 坏或过时的源材料
- 弱分块策略
- 不相关的检索结果
- 缺失支持证据
- 答案超出检索证据范围声称
