# 模板 - Stage 07 评估工具

## 目的

本模板是一个小型评估和回归工具的启动蓝图，围绕一个已存在的 AI 支撑工具、工作流或 agent。

目的不是建排行榜。目的是使行为变更可见。

## 适用场景

- Prompt 回归检查
- 结构化输出有效性检查
- 检索质量检查
- 工具使用场景检查
- 工作流级别通过/失败案例

## 构建目标

- 定义一个可重复的场景集
- 清楚定义接受标准
- 跨版本记录结果
- 在演示或部署前使回归可见

## 建议文件树

```text
stage-07-eval-harness/
  README.md
  ARCHITECTURE.md
  TODO.md
  eval/
    cases/
    results/
  src/
    runner.ts
    assertions.ts
    report.ts
```

## 最低接受标准

- 一个可复用的场景集
- 每种案例类型一条显式通过/失败规则
- 一种结果记录格式
- 一个真实弱案例或回归的例子
- 一条简短笔记，说明在发布或 prompt 变更前如何使用工具

## 扩展想法

- 跨 prompt 版本对比结果
- 跨版本对比检索或工具行为
- 添加简单汇总报告
- 如有需要，后续与 CI 集成
