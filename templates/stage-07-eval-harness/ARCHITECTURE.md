# 架构说明 - Stage 07 评估工具

## 核心理念

评估工具应该使变更可衡量，而不是神秘的。

## 核心组件

1. 场景定义
2. 执行运行器
3. 断言规则或接受检查
4. 结果存储
5. 对比和报告层

## 边界规则

- 场景应代表真实任务或失败类别
- 断言应足够显式，另一个工程师能审查它们
- 結果应能随时间对比
- 评估逻辑应与被测系统分离

## 推荐流程

1. 定义代表性案例
2. 运行目标系统
3. 应用断言或接受检查
4. 记录输出和结果
5. 在适当时与先前运行对比

## 需要计划的失败类别

- Prompt 回归
- 无效结构化输出
- 检索未命中
- 工具使用错误
- 工作流步骤失败
- 运维可见的行为漂移
