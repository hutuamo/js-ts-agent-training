# Stage 05 练习

## 练习原则

每个练习都应该包含至少一个保存的 trace 或运行日志。Agent 行为在它还小的时候必须是可检查的。

## 练习 1 - 手动 agent 循环

目标：让控制流显式化。

任务：

- 实现一个简单的观察-决策-执行-更新循环
- 跟踪步骤计数和终止原因
- 支持最大步数限制
- 打印或持久化发生了什么的小结

进度检查：

- 另一个工程师能说出循环从哪里开始、状态在哪里更新、在哪里停止

## 练习 2 - 工具注册表训练

目标：停止把工具调用当成临时的辅助函数。

任务：

- 用清晰的名称和参数形状定义两三个工具
- 执行前校验参数
- 把工具结果归一化为一致的结构
- 在 trace 中记录工具调用元数据

进度检查：

- 循环能推理工具，而不依赖工具特定的实现细节

## 练习 3 - 工作状态设计

目标：把活跃任务状态和持久会话数据分开。

任务：

- 为当前运行定义一个小状态对象
- 识别哪些字段是临时的，哪些适合持久化
- 每步后更新状态
- 当状态不应该继续传递时，拒绝或清除它

进度检查：

- 你能精确解释哪些状态在轮次之间存活，以及为什么

## 练习 4 - 规划与直接执行对比

目标：决定规划是否增加了价值。

任务：

- 选一个多步任务
- 实现一个直接执行版本
- 再实现一个有显式计划步骤的版本
- 对比复杂度、失败率和 trace 可读性
- 写一段短笔记，说明你会保留哪个版本

进度检查：

- 你能用证据而不是直觉来为规划决策辩护

## 练习 5 - 循环护栏训练

目标：确保循环能安全失败。

任务：

- 加一个最大步数限制
- 加无效工具请求处理
- 加一个不支持任务的拒绝或上报路径
- 加一条路径，在失败时返回部分进展，而不是静默失败

进度检查：

- 你能展示循环在至少三种不同失败条件下如何终止

## 练习 6 - Trace 回顾会话

目标：学会在事后调试 agent 运行。

任务：

- 用多个有代表性的任务运行
- 保存 trace
- 按循环阶段对失败分类
- 识别一个由 prompt 措辞引起的失败，和一个由编排逻辑引起的失败
- 为每个提出一个具体的修复方案

进度检查：

- 你能仅从 trace 调试，而不需要立即重放精确场景

## 练习 7 - 人工交接边界

目标：避免假装自动化永远是对的。

任务：

- 定义一类任务，其中 agent 应该停下来请求确认
- 定义应该交给人类的上下文内容
- 在状态和日志中显式化交接
- 用至少两个场景测试交接

进度检查：

- 交接保留了有用的上下文

## 最低完成要求

至少完成：

- 练习 1、2、3、5
- 一个显式 agent 循环实现
- 一个有护栏和停止条件的版本
- 至少保存了三个运行的 trace 或日志
