# Stage 05 项目

## 项目 1 - 研究助手 Agent

构建一个能用几个受控工具回答有界研究问题的小型 agent。

建议工具：

- 本地笔记查找
- Web 或文档获取包装器（如果环境允许）
- 摘要或提取辅助工具
- 引用格式化器

必需能力：

- 显式 agent 循环
- 带校验参数的工具体系
- 每步 trace 输出
- 分开证据和综合的最终答案

这个项目训练什么：

- 有界研究编排
- 工具使用检查
- 为 Stage 06 的检索密集型工作做准备

验收标准：

- agent 在明确的完成规则后停止
- trace 显示哪些工具被调用了以及为什么
- 不支持或歧义的查询被显式拒绝或上报

## 项目 2 - 多步本地工作流 Agent

构建一个能协调本地工具完成一个后端或 CLI 任务的 agent。

例子：

- 检查日志并产生事件摘要
- 收集仓库元数据并产生审查清单
- 运行带中间校验的本地分析流水线

必需能力：

- 有界步数
- 结构化中间状态
- 至少一个副作用感知工具分类
- 带步骤结果的 trace 日志

这个项目训练什么：

- 本地工具编排
- 跨步骤的状态管理
- 决定 agent 行为在哪里停止、确定性工作流在哪里开始

验收标准：

- 系统不把副作用隐藏在模糊的工具描述后面
- 状态更新是可见的且可审查的
- 最终输出可以追溯到具体的工具结果

## 项目 3 - 带可审查 Trace 的工具调用助手

构建一个强调可检查性的小型运维任务助手。

例子：

- 支持工单分类助手
- 仓库维护助手
- 运维摘要助手

必需能力：

- 短生命周期会话状态
- Trace 持久化或可审查的运行记录
- 无效工具请求处理
- 显式停止和上报规则

这个项目训练什么：

- 审查友好的 agent 设计
- 运维调试习惯
- 为 Stage 07 评估和加固工作打基础

验收标准：

- 审查者能仅从日志或 trace 工件重建运行过程
- 助手不会在不清楚的任务上无限继续
- 失败结果被分类，而不是被collapse成一个通用错误

## 推荐完成顺序

1. 研究助手 Agent
2. 多步本地工作流 Agent
3. 带可审查 Trace 的工具调用助手
