# Stage 06 项目

## 项目 1 - 文档 QA 助手

在有界文档集上构建一个检索支撑的助手。

适合的语料：

- 内部笔记
- 技术设计文档
- 事件报告
- 文档快照

必需能力：

- 摄入流水线
- 分块和元数据策略
- 可独立检查的检索步骤
- 带引用或来源参考的有据可查的答案

这个项目训练什么：

- 端到端检索设计
- provenance 感知的答案生成
- 为生产评估工作做准备

验收标准：

- 答案引用它们所依赖的检索证据
- 不被支持的提问被显式拒绝或标记
- 检索结果可以独立于最终输出被检查

## 项目 2 - 研究笔记本生成器

构建一个收集材料、提取要点并产生结构化笔记本或报告的工作流。

必需能力：

- 跨先前材料的检索
- 中间摘要或提取的facts
- 源支持内容与综合内容的显式区分
- 可独立重新运行的流水线阶段

这个项目训练什么：

- 工作流组合
- 中间产物设计
- 研究任务的记忆和检索边界

验收标准：

- 工作流比单个自由形式 agent 循环更可检查
- 每个阶段产生可审查的产物
- 最终输出包含回溯到源材料的参考

## 项目 3 - 检索支撑工作流运行器

构建一个使用检索来支持重复运维任务的后端或 CLI 工作流。

例子：

- 事件分类助手
- 支持响应草稿流水线
- 变更审查上下文组装器

必需能力：

- 清晰的工作流阶段
- 检索支撑的上下文组装
- 带类型的中间数据
- 需要时的显式交接或批准点

这个项目训练什么：

- 运维风格工作流中的实用检索
- 确定性编排内的有界模型使用
- 为 Stage 07 的生产加固做准备

验收标准：

- 每个工作流阶段有清晰的契约
- 上下文组装可重现
- 系统能解释哪些检索输入实际上影响了最终输出

## 推荐完成顺序

1. 文档 QA 助手
2. 研究笔记本生成器
3. 检索支撑工作流运行器
