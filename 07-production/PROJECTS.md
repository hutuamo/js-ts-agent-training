# Stage 07 项目

## 项目 1 - 加固已有 Agent 或工作流

取一个 Stage 04、05 或 06 的项目，为重复使用加固它。

必需能力：

- 边界校验
- 结构化日志和运行 ID
- 显式重试和超时策略
- 失败分类
- 面向运维者的文档

这个项目训练什么：

- 把原型变成可维护系统
- 纪律化的失败处理
- 跨阶段可靠性实践的整合

验收标准：

- 关键失败路径在日志和文档中可见
- 输入输出在外部边界被校验
- 另一个工程师能通过阅读项目和笔记解释运行时行为

## 项目 2 - 评估套件和基准包

为一个 AI 支撑项目构建可复用的评估工具。

必需能力：

- 有代表性的场景集
- 预期结果或接受属性
- prompt、检索或工具使用覆盖
- 跨版本的結果记录

这个项目训练什么：

- 回归纪律
- 可衡量的改进工作
- 在 AI 系统行为上安全迭代

验收标准：

- 该套件能抓到至少一个真实回归或弱案例
- 結果以支持随时间对比的方式存储
- 通过/失败规则足够具体，另一个工程师也能用

## 项目 3 - 最小可部署服务或 Worker

把一个项目打包成能在你的开发 shell 之外重复运行的东西。

例子：

- 一个小型 API 服务
- 一个定时 worker
- 带部署笔记的命令运行器

必需能力：

- 启动校验
- 环境特定配置处理
- 运维文档
- 安全关闭或失败行为

这个项目训练什么：

- 部署形态决策
- 运维就绪
- 把本地开发工作转化为可重复执行

验收标准：

- 配置无效时启动明确失败
- 运行时日志有用且安全
- 存在基本的发布和回滚故事

## 推荐完成顺序

1. 加固已有 Agent 或工作流
2. 评估套件和基准包
3. 最小可部署服务或 Worker
