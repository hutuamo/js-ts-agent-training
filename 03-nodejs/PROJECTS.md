# Stage 03 项目

## 项目 1 - 多命令工具集

做一个带多个子命令的 Node.js CLI。

建议命令：

- 查看文件信息
- 汇总 JSON 或日志
- 调用一个 HTTP endpoint
- 执行一个本地自动化任务

必需能力：

- 参数解析
- 配置加载
- 结构化错误处理
- 命令处理逻辑与公共辅助模块的清晰分离

这个项目训练什么：

- CLI 架构
- 工程纪律
- 可复用的后端工具结构

验收标准：

- 每个命令都有清楚契约
- 失败路径有明确退出行为
- 日志与用户输出被有意识地区分

## 项目 2 - 可靠的 API 集成服务

做一个小型服务或 worker 风格脚本，集成一个外部 API。

要求：

- 带类型的配置
- 重试与超时
- 归一化输出
- 对尝试与失败有日志记录
- 如果一次处理多个请求，要支持局部失败处理

这个项目训练什么：

- 稳定 I/O 行为
- 服务式控制流
- 为模型提供商集成做准备

验收标准：

- transport error 与业务错误被区分
- timeout 和 retry 策略写得清楚
- 仅通过日志也能推理服务行为

## 项目 3 - 本地自动化脚本套件

做一小组脚本，用于包装本地工具或进程。

例子：

- 仓库检查辅助工具
- 代码搜索与摘要工作流
- 文件转换流水线
- 带结构化结果输出的批命令执行器

这个项目训练什么：

- 子进程处理
- 任务编排
- 在 Node 环境中控制外部能力

验收标准：

- 命令执行结果是结构化的
- 局部失败不会拖垮无关任务
- 对同一套输入重复执行时行为稳定

## 推荐完成顺序

1. 多命令工具集
2. 可靠的 API 集成服务
3. 本地自动化脚本套件
