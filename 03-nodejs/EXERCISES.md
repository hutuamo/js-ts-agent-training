# Stage 03 练习

## 练习原则

本阶段所有练习都至少要包含一个失败场景检查。

最低标准不是“成功跑过一次”，而是“在常见失败下行为仍然可预测”。

## 练习 1 - 文件遍历器

目标：把本地文件系统处理写干净。

任务：

- 遍历目录树
- 按扩展名或自定义条件筛选文件
- 输出文件数量与总大小摘要
- 对不可读文件给出清楚警告，而不是直接崩掉

通过标准：

- 脚本能区分“遍历逻辑”和“汇报逻辑”

## 练习 2 - 配置加载器

目标：不要把配置假设散落在代码里。

任务：

- 从环境变量和可选配置文件中加载配置
- 校验必需字段
- 对启动时的配置错误给出清晰报错
- 避免把密钥打进普通日志

通过标准：

- 当配置缺失或不合法时，系统会在启动阶段快速失败

## 练习 3 - 超时与重试包装器

目标：默认把外部 I/O 当成不可靠。

任务：

- 给异步操作加 timeout 支持
- 增加有上限的重试与 backoff
- 定义哪些错误值得重试
- 记录每次尝试和最终失败

通过标准：

- 你能解释为什么有些失败不该重试

## 练习 4 - HTTP client 加固练习

目标：为模型和工具 API 集成做准备。

任务：

- 调用一个 HTTP endpoint
- 处理非 2xx 响应
- 处理非法 JSON
- 处理 timeout 和网络异常
- 为下游消费者输出归一化结构

通过标准：

- 下游代码不需要知道 transport 层细节

## 练习 5 - 流式处理练习

目标：学会在输入变大时合理使用 stream，而不是默认一次性读入。

任务：

- 处理一个大文本文件或模拟大输入流
- 增量统计行数或提取记录
- 对比整文件读取与流式处理
- 写一小段说明，解释各自取舍

通过标准：

- 你能解释什么时候 stream 真值得，什么时候只是增加复杂度

## 练习 6 - 子进程包装器

目标：安全集成本地外部工具。

任务：

- 运行一个外部命令
- 捕获 stdout 和 stderr
- 处理非零退出码
- 向调用方返回结构化结果

通过标准：

- 你的包装器不会把“命令失败”和“包装器本身失败”混为一谈

## 练习 7 - 并发控制练习

目标：学会同时管理多个异步任务。

任务：

- 处理一组异步任务
- 实现固定并发上限
- 记录成功数与失败数
- 对比顺序执行和并发执行的运行表现

通过标准：

- 你能解释为什么“不设上限并发”往往不是默认正确选项

## 最低完成要求

至少完成：

- 练习 2、3、4、7
- 一个子进程集成练习
- 一个包含显式失败处理的文件处理练习
