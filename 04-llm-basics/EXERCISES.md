# Stage 04 练习

## 练习原则

每个练习都应该产出保存下来的案例或笔记。你在积累后续评估和生产化加固的原始材料。不要依赖记忆。

## 练习 1 - 最小模型调用包装器

目标：把 LLM 提供商当成普通外部服务来对待。

任务：

- 创建一个小型包装器，发送一个 prompt，返回归一化文本输出
- 从配置加载凭证，而不是硬编码
- 记录可保留的请求元数据
- 区分 transport 失败、提供商拒绝和应用级解析失败

进度检查：

- 其他代码不依赖原始提供商响应对象

## 练习 2 - Prompt 契约训练

目标：把 prompt 写成工程契约，而不是模糊指令。

任务：

- 选一个具体后端任务，如摘要、提取或分类
- 写一个定义了任务、约束和输出期望的 prompt
- 用多个有代表性的输入运行它
- 记录 prompt 在哪里失败或变得模糊
- 用最小改动修订 prompt

进度检查：

- 你能解释哪些改动提升了可靠性，哪些是不必要的

## 练习 3 - 带校验的结构化 JSON 输出

目标：确保模型输出默认不被信任。

任务：

- 为一个实际任务定义目标 JSON 形状
- 让模型产生该形状
- 在运行时校验响应
- 显式拒绝或修复无效响应
- 记录至少三个无效或不完美的案例

进度检查：

- 下游逻辑只接收经过校验的数据

## 练习 4 - 流式响应训练

目标：理解流式何时有帮助，何时会让正确性更复杂。

任务：

- 实现一个流式响应路径和一个缓冲响应路径
- 捕获部分 chunk，不假设它们能组成有效最终输出
- 对比感知延迟和实现复杂度
- 写一段短文，说明你会为 CLI 选择哪条路径，以及为什么

进度检查：

- 你能说出流式响应在哪个精确时刻变得可以安全消费

## 练习 5 - 单次工具调用往返

目标：加入一个受控的动作边界。

任务：

- 定义一个用途单一的工具
- 清楚暴露其名称、描述和参数 schema
- 执行前校验工具参数
- 归一化工具结果再返回给模型
- 在一次工具执行和一次后续模型响应后停止

进度检查：

- 无效工具参数作为校验错误失败，而不是隐藏的运行时行为

## 练习 6 - 成本与延迟测量训练

目标：停止把模型使用当成免费和即时的。

任务：

- 跨多个请求测量一个任务的延迟
- 如果有条件，对比至少两个 prompt 变体或模型选择
- 记录输入大小、输出大小和观察到的成本指标
- 写一段短笔记，说明你会先优化什么

进度检查：

- 你能为至少一个延迟或成本权衡说出具体数字，而不是猜测

## 练习 7 - 模型边界记录

目标：建立你自己的失败模式知识库。

任务：

- 汇总你在练习和项目中观察到的所有模型失效模式
- 按类型分类（格式错误、prompt 敏感性、截断、不完整输出等）
- 为每种类型记录你采取的缓解措施
- 说明哪些类型的失败是最难完全消除的

进度检查：

- 你能说出一个新失败是哪种类型，而不是把它归为“模型抽风”

## 最低完成要求

至少完成：

- 练习 1、2、3、5
- 一个带校验的结构化输出练习
- 至少一个流式路径的实现或对比分析
- 一份你自己的模型失效模式笔记
